# 需求规格说明书（更新版）

## 1. 引言

### 1.1 目的
本文档定义Mac数据库管理工具的功能需求，目标是1:1复刻Navicat，优先支持MySQL，用于内部使用。

### 1.2 范围
开发macOS原生数据库管理工具，1:1复刻Navicat的界面和功能，优先支持MySQL数据库。

### 1.3 关键要求
- 1:1复刻Navicat界面设计
- 优先支持MySQL
- 构建dmg安装包
- 内部使用，不考虑法律风险

## 2. 总体描述

### 2.1 产品前景
1:1复刻Navicat，提供相同的用户体验，但为macOS原生应用，性能更好。

### 2.2 用户特征
- 开发人员
- 数据库管理员
- 数据分析师
- 团队内部成员

### 2.3 运行环境
- **操作系统**: macOS 10.15+
- **数据库**: MySQL 5.7+
- **网络**: 需要网络连接远程MySQL

## 3. 功能需求

### 3.1 连接管理（1:1复刻Navicat）
#### 3.1.1 连接对话框
- **FR-001**: 复刻Navicat的新建连接对话框
- **FR-002**: 连接名、主机、端口、用户名、密码字段
- **FR-003**: 测试连接按钮
- **FR-004**: 连接颜色标记
- **FR-005**: HTTP隧道选项（可选）

#### 3.1.2 连接列表
- **FR-006**: 左侧连接树形列表
- **FR-007**: 连接分组功能
- **FR-008**: 右键菜单（打开、编辑、删除、复制）
- **FR-009**: 连接状态图标

#### 3.1.3 连接配置
- **FR-010**: SSL配置
- **FR-011**: SSH隧道配置
- **FR-012**: 高级设置（字符集、超时等）

### 3.2 数据库管理（1:1复刻Navicat）
#### 3.2.1 对象浏览器
- **FR-013**: 左侧树形结构显示数据库对象
- **FR-014**: 表、视图、存储过程、函数分类
- **FR-015**: 对象搜索功能
- **FR-016**: 对象过滤功能

#### 3.2.2 表管理
- **FR-017**: 表设计器（字段、索引、外键、约束）
- **FR-018**: 表数据网格视图
- **FR-019**: 表结构SQL预览
- **FR-020**: 表导入导出向导

#### 3.2.3 数据编辑
- **FR-021**: 网格形式编辑数据
- **FR-022**: 添加、删除、复制行
- **FR-023**: 数据筛选和排序
- **FR-024**: 数据搜索
- **FR-025**: 批量编辑

### 3.3 SQL查询（1:1复刻Navicat）
#### 3.3.1 查询编辑器
- **FR-026**: SQL语法高亮
- **FR-027**: 自动补全（表名、字段名、关键字）
- **FR-028**: 行号显示
- **FR-029**: 查询标签页
- **FR-030**: 查询历史记录

#### 3.3.2 查询执行

#### 3.3.3 结果显示

- **FR-036**: 结果导出（CSV、JSON、Excel）
- **FR-037**: 结果筛选和排序
- **FR-038**: 结果统计信息

### 3.4 数据导入导出（1:1复刻Navicat）
#### 3.4.1 导入向导
- **FR-039**: 支持CSV、JSON、Excel、SQL文件导入

- **FR-041**: 数据预览

#### 3.4.2 导出向导

### 3.5 数据库工具（1:1复刻Navicat）
#### 3.5.1 备份恢复
- **FR-047**: 数据库备份向导
- **FR-048**: 数据库恢复向导

#### 3.5.2 数据传输
- **FR-050**: 数据库间数据传输

- **FR-052**: 传输日志

### 3.6 界面设计（1:1复刻Navicat）
#### 3.6.1 主窗口布局

- **FR-054**: 右侧工作区（标签页）

- **FR-056**: 底部状态栏

#### 3.6.2 工具栏

#### 3.6.3 标签页

## 4. 非功能需求

### 4.1 性能需求
- **NFR-001**: 应用启动时间 < 2秒
- **NFR-002**: 连接MySQL时间 < 3秒
- **NFR-003**: 查询响应时间 < 1秒（本地）
- **NFR-004**: 支持10万行数据流畅显示
- **NFR-005**: 内存占用 < 300MB

### 4.2 界面需求
- **NFR-006**: 1:1复刻Navicat界面设计
- **NFR-007**: 支持Dark Mode和Light Mode
- **NFR-008**: 支持Retina显示屏
- **NFR-009**: 响应式布局

### 4.3 可靠性需求
- **NFR-010**: 应用崩溃率 < 0.01%
- **NFR-011**: 数据丢失率为0
- **NFR-012**: 自动保存查询历史

### 4.4 兼容性需求
- **NFR-013**: 支持Intel和Apple Silicon
- **NFR-014**: 支持macOS 10.15+
- **NFR-015**: 支持MySQL 5.7+

## 5. 外部接口需求

### 5.1 用户接口
- **UI-001**: 精确复刻Navicat的界面设计
- **UI-002**: 相同的菜单结构和快捷键
- **UI-003**: 相同的对话框布局
- **UI-004**: 相同的图标和颜色方案

### 5.2 硬件接口
- **HW-001**: 显示器（支持Retina）
- **HW-002**: 键盘鼠标
- **HW-003**: 网络接口

### 5.3 软件接口
- **SW-001**: macOS系统API
- **SW-002**: MySQL客户端库
- **SW-003**: 文件系统API

## 6. 系统特性

### 6.1 1:1复刻
- 界面布局完全一致
- 交互逻辑完全一致
- 功能命名完全一致

### 6.2 MySQL优先
- 优先实现MySQL所有功能
- 其他数据库后续扩展

### 6.3 macOS原生
- 使用Swift/SwiftUI开发
- 符合macOS设计规范
- 性能优化
