# 技术架构设计文档

## 1. 架构概述

### 1.1 设计目标
- 1:1复刻Navicat界面和交互
- 优先支持MySQL数据库
- 构建dmg安装包
- macOS原生应用，性能优秀

### 1.2 架构模式
采用MVVM（Model-View-ViewModel）架构：
- **Model**: 数据模型和业务逻辑
- **View**: SwiftUI视图
- **ViewModel**: 视图状态和业务逻辑

### 1.3 技术栈
| 层次 | 技术选择 | 说明 |
|------|----------|------|
| UI层 | SwiftUI | macOS原生UI框架 |
| 业务逻辑层 | Swift | 核心业务逻辑 |
| 数据层 | MySQLNIO | MySQL连接库 |
| 网络层 | SwiftNIO | 异步网络框架 |
| 持久化层 | UserDefaults, Keychain | 配置和密码存储 |

## 3. 核心模块设计

### 3.1 MySQL连接模块
#### 3.1.1 技术选择
- **MySQLNIO**: 基于SwiftNIO的MySQL客户端
- **异步编程**: 使用async/await
- **连接池**: 管理多个数据库连接

#### 3.1.2 核心类
```swift
// MySQLService.swift
class MySQLService {
    // 连接配置
    struct ConnectionConfig {
        let host: String
        let port: Int
        let username: String
        let password: String
        let database: String?
        let ssl: Bool
    }
    
    // 连接管理
    func connect(config: ConnectionConfig) async throws -> Connection
    func disconnect(connection: Connection) async throws
    func testConnection(config: ConnectionConfig) async throws -> Bool
    
    // 查询执行
    func execute(query: String, on connection: Connection) async throws -> QueryResult
    func executeAsync(query: String, on connection: Connection) async throws -> AsyncStream<QueryResult>
}
```

### 3.2 连接管理模块
#### 3.2.1 功能
- 连接配置的增删改查
- 连接状态管理

// ConnectionManager.swift
class ConnectionManager {

## 4. 数据流设计

### 4.1 连接流程
```
用户输入连接信息 -> 测试连接 -> 保存连接配置 -> 建立连接 -> 显示数据库对象
```

### 4.2 查询流程
```
输入SQL -> 执行查询 -> 显示结果 -> 编辑数据 -> 保存更改
```

### 4.3 导入流程
```
选择文件 -> 预览数据 -> 映射字段 -> 导入数据 -> 显示结果
```

## 8. 技术风险与应对

### 8.1 MySQL连接库选择
- **风险**: MySQLNIO可能不够成熟
- **应对**: 准备备选方案（如直接使用C库封装）

### 8.2 界面复刻难度
- **风险**: 完全复刻Navicat界面困难
- **应对**: 使用截图对比，逐步调整

### 8.3 性能问题
- **风险**: 大数据量显示卡顿
- **应对**: 使用懒加载、分页显示
