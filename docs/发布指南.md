# NavicatMac 版本发布指南

## 自动化发布流程

本项目提供了自动化发布脚本，可以快速完成版本发布。

### 使用发布脚本

```bash
# 发布新版本
./scripts/release.sh -v 0.2.0 -m "新增功能描述"

# 模拟运行（不执行实际操作）
./scripts/release.sh -v 0.2.0 -m "新增功能描述" -d

# 强制执行（跳过确认）
./scripts/release.sh -v 0.2.0 -m "新增功能描述" -f
```

### 发布脚本功能

1. **验证版本号格式** - 确保版本号为 X.Y.Z 格式
2. **检查Git状态** - 确保没有未提交的更改
3. **更新版本号** - 更新Package.swift和Info.plist中的版本号
4. **更新CHANGELOG.md** - 自动更新变更日志
5. **构建项目** - 构建Release版本和dmg包
6. **创建Git标签** - 创建带注释的Git标签
7. **推送到远程仓库** - 推送代码和标签到GitHub
8. **创建GitHub Release** - 自动创建Release（需要GitHub Token）

## 手动发布流程

如果需要手动发布，请按照以下步骤操作：

### 1. 准备发布

```bash
# 检查Git状态
git status

# 确保所有更改已提交
git add .
git commit -m "feat: 新功能描述"
```

### 2. 更新版本号

编辑以下文件，更新版本号：
- `Package.swift` - 更新 `let version = "0.2.0"`
- `NavicatMac/Info.plist` - 更新 `CFBundleShortVersionString`

### 3. 更新CHANGELOG.md

在 `CHANGELOG.md` 中添加新版本信息：

```markdown
## [0.2.0] - 2026-05-12

### 新增
- 新增功能1
- 新增功能2

### 变更
- 变更内容

### 修复
- 修复问题
```

### 4. 构建项目

```bash
# 构建Release版本
make release

# 打包dmg
make package
```

### 5. 创建Git标签

```bash
# 添加所有更改
git add .
git commit -m "release: v0.2.0 - 版本描述"

# 创建标签
git tag -a v0.2.0 -m "Release v0.2.0: 版本描述"

# 推送到远程仓库
git push origin main
git push origin v0.2.0
```

### 6. 创建GitHub Release

1. 访问 https://github.com/yxgr-copilot/navicat-mac/releases/new
2. 选择标签: `v0.2.0`
3. 标题: `Release v0.2.0`
4. 描述: 复制CHANGELOG.md中的内容
5. 上传 `build/dmg/NavicatMac-20260512.dmg` 文件
6. 点击 "Publish release"

## GitHub Token配置

为了使用自动化发布脚本，需要配置GitHub Token：

### 1. 创建GitHub Token

1. 访问 https://github.com/settings/tokens
2. 点击 "Generate new token"
3. 选择权限: `repo` (完整仓库访问权限)
4. 生成并保存Token

### 2. 设置环境变量

```bash
# 临时设置
export GITHUB_TOKEN="your_token_here"

# 永久设置（添加到 ~/.zshrc 或 ~/.bashrc）
echo 'export GITHUB_TOKEN="your_token_here"' >> ~/.zshrc
source ~/.zshrc
```

### 3. 验证配置

```bash
# 测试GitHub API访问
curl -H "Authorization: token $GITHUB_TOKEN" https://api.github.com/user
```

## 版本号规范

本项目遵循 [语义化版本控制](https://semver.org/lang/zh-CN/)：

- **主版本号 (MAJOR)**: 不兼容的API修改
- **次版本号 (MINOR)**: 向下兼容的功能性新增
- **修订号 (PATCH)**: 向下兼容的问题修正

### 示例

- `0.1.0` -> `0.1.1`: 修复bug
- `0.1.0` -> `0.2.0`: 新增功能
- `0.1.0` -> `1.0.0`: 正式发布版本

## 变更日志规范

变更日志遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/) 规范：

### 变更类型

- **新增 (Added)**: 新功能
- **变更 (Changed)**: 对现有功能的变更
- **弃用 (Deprecated)**: 不建议使用的功能
- **移除 (Removed)**: 移除的功能
- **修复 (Fixed)**: bug修复
- **安全 (Security)**: 安全相关的变更

### 示例

```markdown
## [0.2.0] - 2026-05-12

### 新增
- 新增PostgreSQL数据库支持
- 新增数据导出功能

### 变更
- 优化查询编辑器性能
- 更新UI界面设计

### 修复
- 修复连接超时问题
- 修复数据导入错误
```

## 常见问题

### Q: 如何回滚版本？

A: 如果需要回滚版本：

```bash
# 删除本地标签
git tag -d v0.2.0

# 删除远程标签
git push origin :refs/tags/v0.2.0

# 删除GitHub Release（在GitHub网页上操作）

# 回退代码
git revert HEAD
git push origin main
```

### Q: 如何修改已发布的Release？

A: 在GitHub网页上编辑Release，更新描述或上传新的dmg文件。

### Q: 如何设置自动发布？

A: 可以使用GitHub Actions实现自动发布。在 `.github/workflows/release.yml` 中配置自动化工作流。
